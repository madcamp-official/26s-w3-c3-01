# 🎱 3쿠션 당구 데이터 파이프라인 — 구축 현황

유튜브 당구 영상 링크 → 매 턴의 **공 위치(0~1), 수구, 성공 여부, 다음 배치**를 자동 추출해 PostgreSQL에 저장하는 시스템.
캠퍼스망이 클라우드 DB 포트를 막는 문제까지 우회해 **완성 직전** 단계.

- ✅ 추출·저장·적재 검증 완료
- ⏳ 남은 것: cron 자동화 1가지

---

## 01. 전체 구조

연산은 맥에서 무료로(GPU 가속), 저장은 AWS에서. 맥은 S3에만 올리고, DB 적재는 AWS 안에서 처리.

```
[맥 · 캠퍼스망]                         [AWS 서울]
YOLO로 영상 분석            업로드          S3 버킷              읽기         EC2 로더 → RDS
→ 턴 데이터 추출    ──────────────▶   billiard-turns-   ──────────▶   t3.micro가 S3 읽어
                    (443, 통과됨)     mcamp3w           (VPC 내부)    PostgreSQL에 적재
```

---

## 02. 도중에 부딪힌 진짜 문제 — 그리고 우회

### 🚧 캠퍼스망이 AWS로 나가는 5432(DB 포트)를 차단

맥에서 RDS에 직접 붙으려니 계속 타임아웃. 보안그룹을 전체 개방(`0.0.0.0/0`)해도 안 됐고,
아래 진단으로 **원인이 AWS가 아니라 학교 네트워크**임이 확정됨.

| 테스트 | 결과 | 의미 |
|---|---|---|
| 외부 공개 PostgreSQL 5432 (유럽) | ✅ 성공 | 맥의 5432 송신 자체는 됨 |
| AWS 서울 443 (S3) | ✅ 성공 | AWS엔 443으로는 닿음 |
| AWS 서울 5432 (RDS) | ❌ 타임아웃 | AWS로 가는 5432만 막힘 |

**해결:** 맥은 **443으로 S3에만 업로드**(통과됨), RDS 적재는 **같은 VPC 안의 EC2**가 담당 →
막힌 구간을 아예 안 지나감. 공교롭게 앞서 설계한 S3 경유 구조가 그대로 정답이 됐음.

## 03. 만든 것 — 코드

### 추출 파이프라인 (맥)

| 파일 | 역할 |
|---|---|
| `extract_turns.py` | 턴 페어링 · 수구 식별 · 성공 판정 |
| `detect_pipeline.py` | 당구대 검출 (파랑·초록·회색 천 대응) |
| `detect_video.py` | 탑뷰 판별 (경계 접촉 필터) |
| `process_queue.py` | 큐 처리 + S3 업로드 + 영상 삭제 |
| `best_3cls.pt` | YOLO 공 탐지 모델 |

### 적재 로더 (AWS)

| 파일 / 객체 | 역할 |
|---|---|
| `db/load_to_db.py` | S3 → PostgreSQL upsert (중복 안 생김) |
| `db/schema.sql` | 테이블 자동 생성 |
| `billiard_turns` | 턴별 데이터 테이블 |
| `billiard_ingest_log` | 영상별 적재 이력 |

## 04. 만든 것 — AWS 리소스 (전부 서울 · 무료 티어)

| 리소스 | 이름 / 값 |
|---|---|
| S3 | `billiard-turns-mcamp3w` (결과 + 코드) |

| EC2 | `billiard-loader` · t3.micro · 무료 티어 |
| IAM 역할 | `billiard-loader-role` (S3 읽기) |
| 보안그룹 | RDS는 VPC 내부(`172.31.0.0/16`)만 허용 |

---

## 05. 검증된 것

- ✅ **영상 추출** — 3개 영상(PBA 팀리그·챔피언십·LPBA)에서 **총 24턴** 추출, 천 색 3종(파랑·초록·회색) 모두 처리
  · 성공/실패 판정이 점수판·턴 교대와 일치
- ✅ **맥 → S3 업로드** — 캠퍼스망(443)에서 결과 파일 정상 업로드
- ✅ **EC2 → RDS 적재** — S3의 24턴을 PostgreSQL에 upsert 성공 · 재실행해도 중복 없음

---

## 06. 남은 것 — cron 자동화

**딱 하나 남음:** EC2에 `load_to_db.py --all`을 주기(예: 10분)마다 돌리는 **cron**만 걸면,
S3에 새 결과가 올라올 때마다 자동으로 DB에 반영됨. (맥 쪽 추출 자동 실행도 원하면 이어서 설정.)

### 보안 마무리 (같이 하면 좋음)

- RDS 보안그룹에서 맥 IP·임시(`0.0.0.0/0`) 규칙 정리 → VPC 내부만 남기기
- EC2 SSH(22) 소스를 `0.0.0.0/0` → 내 IP로 좁히기

## 데이터 스키마 참고 (`billiard_turns`)

| 컬럼 | 설명 |
|---|---|
| `video_id`, `turn` | 영상 id + 턴 순번 (복합 기본키) |
| `epoch` | 연속 구간(클립) 번호 — 값이 튀면 카메라 컷 경계 |
| `shooter` | 수구: `white` / `yellow` / `red` |
| `success` | 3쿠션 성공 여부 (판정 불가 시 `NULL`) |
| `success_method`, `coverage`, `cushions_before_2nd`, `hits` | 성공 판정 근거 |
| `before_pos`, `after_pos` | 샷 직전/이후 3구 좌표 (JSONB, 0~1 정규화) |
| `after_source` | `settled`(정지 확인) / `last_seen`(마지막 관측) |
| `frame_start`, `frame_end`, `time_start_s`, `time_end_s` | 구간 정보 |
