# 📐 턴 데이터 추출 기준 (Extraction Criteria)

> `src/extract_turns.py` + `src/scoreboard.py` 가 영상에서 턴 데이터를 뽑는 **판정 기준 명세**.
> **대원칙: 라벨(수구·성공·뱅크샷·턴 교대)은 전부 방송 점수판에서 얻는다.**
> 영상(비전)은 "공이 어디에 있고 언제 멈췄는가"(좌표·타이밍)만 담당한다.

---

## 0. 점수판 우선 원칙 — 항목별 근거 소스

| 추출 항목 | 근거 | 방식 |
|---|---|---|
| **수구(shooter)** | 🟡 점수판 | 이닝 원형에 **숫자가 떠 있는 색** = 지금 치는 선수 |
| **턴 교대** | 🟡 점수판 | 샷 실패 시 숫자가 **상대 색 원으로 이동**(상대 원에 0부터 표시) → 색 전환 감지 |
| **성공/실패** | 🟡 점수판 | 샷 창 안에서 수구 색 점수 **+1/+2 스텝** 유무 (총점 박스·이닝 원형 이중 근거) |
| **뱅크샷** | 🟡 점수판 | 득점 스텝이 **+2** 면 `bank_shot=true` |
| **리플레이 걸러내기** | 🟡 점수판 | 샷 구간 점수판 **노출 비율 < 30%** → 리플레이 유령 턴으로 폐기 |
| 공 위치(before/after) | 🎥 영상 | 점수판으로는 알 수 없음 — 탑뷰 프레임 YOLO 검출 |
| 샷 시작/종료 타이밍 | 🎥 영상 | 정지 배치 → 이탈 감지 (점수판은 시점 해상도가 없음) |
| 수구 (폴백) | 🎥 영상 | 점수판 판독 불가 구간만: 가장 먼저 움직인 공 (`shooter_source: "motion"`) |

> **점수판 판정(`method: "scoreboard"`)을 받지 못한 턴은 최종 결과에서 폐기**되고
> DB 로더(`db/load_to_db.py`)도 scoreboard 판정 턴만 적재한다. 즉 DB의 모든 라벨은 점수판 근거다.

## 1. 점수판 읽기 방식 (`src/scoreboard.py`)

PBA 점수판 구조 (예: 32강 화면):

```
 ┌────────────────────────────────┐
 │  산체스   [14] (🟡  )  ≡       │   ← 총점 박스 14, 원형: 숫자 없음 = 대기
 │  김재근   [10] (⚪ 1)  ≡       │   ← 총점 박스 10, 원형: 1 = 현재 이닝 1득점 중 (수구=백)
 └────────────────────────────────┘
```

- **총점 박스**: 각 선수 누적 점수 (OCR)
- **이닝 원형**: 현재 치는 선수의 원에만 숫자(이번 이닝 득점)가 뜬다.
  **숫자가 떠 있는 원의 색 = 수구.** 실패로 턴이 넘어가면 숫자가 상대 원으로 이동(0부터).
- 판독 신뢰성 장치:
  - **2회 연속 동일 판독**일 때만 확정 (한 프레임 오독 무시)
  - 두 원이 **둘 다 숫자거나 둘 다 없으면 "모호"** → 보류
  - 총점 40 초과 판독은 오독으로 폐기 (`SCORE_MAX`)
  - 샘플 주기: 처리 프레임 5개마다, **탑뷰가 아니어도**(클로즈업에도 점수판은 떠 있음)

---

## 2. 프레임 사용/폐기 기준

```
모든 프레임 ─┬─ 점수판 OCR (5프레임마다, 화면 종류 무관)
             │
             ├─ [관문1] 탑뷰 형태? (가로 2:1 · 수평 변 · 화면 12~80% · 경계 미접촉)
             ├─ [관문2] 기준 꼭짓점(초기 탑뷰 8프레임 중앙값 합의)과 90px 이내?
             │     실패 → 클로즈업/컷 → 공 검출 없이 버림
             └─ [관문3] YOLO 공 검출 — conf 0.45 미만 버림, 색깔별 최고 1개만
```

| 폐기 규칙 | 기준 |
|---|---|
| 카메라 컷/클로즈업 | 관문1·2 실패 프레임 (공 검출 안 함) |
| 편집 점프 컷 | 공이 프레임당 0.3m(9m/s) 이상 "순간이동" → 진행 중 샷 폐기 + 리셋 |
| 클립 경계 | 탑뷰 4초 이상 끊김 → 리셋, `epoch` 증가 |
| 저신뢰 탐지 | YOLO conf < 0.45 |

## 3. 턴(샷) 분할 — 영상 담당

상태머신 `정지 → 샷 → 정지`:

- **정지 배치**: 세 공 각각 최근 12개 관측 중 8개 이상이 0.008(≈2cm) 범위 + 1.5초 내 관측. 좌표는 중앙값 → `before_pos`
- **샷 시작**: 어느 공이 정지 위치에서 **6cm 이상 이탈을 2회 연속** 관측
- **샷 종료**: 세 공 재정지 → `after_pos` (`after_source: settled`).
  컷/영상 종료로 못 본 경우 마지막 관측 좌표 (`last_seen`)

## 4. 성공/실패/뱅크샷 판정 — 점수판 담당

**판정 창 = [이 샷 시작, 다음 샷 시작)** — 득점은 다음 샷 전에 점수판에 반영된다.

| 판정 | 기준 |
|---|---|
| **성공** | 창 안에서 수구 색의 ① 총점 박스 +1/+2 스텝 **또는** ② 이닝 원형 +1/+2 스텝 |
| **뱅크샷** | 그 스텝이 +2 (`bank_shot=true`) |
| **실패** | 창 안에 스텝 없음 — 단 **창 끝까지 점수판 판독이 이어진 경우에만** 확정 (마지막 턴은 샷 종료 +3초 여유). 판독 끊김 = 실패가 아니라 **판정 보류** |
| 보류→폐기 | 점수가 이상하게 튄 창(하이라이트 컷 등), 판독 공백 |

- 이닝 원형 스텝은 **같은 색 연속 이벤트끼리만** 비교 — 턴 교대로 0 리셋되는 것은 득점으로 치지 않음
- 창에 스텝이 여럿이면 **첫 스텝**이 이 턴의 득점 (뒤 스텝은 놓친 턴의 것)
- 궤적(쿠션 카운트) 판정 결과는 `success_detail.traj_success` 로 **QA 비교용으로만** 보존

### 최종 필터 (저장 전)
1. **리플레이 유령 턴 삭제** — 샷 구간 점수판 노출 < 30%
2. **scoreboard 판정 못 받은 턴 삭제** — 남은 턴만 1부터 재번호
3. DB 로더도 `method="scoreboard"` 만 적재 (이중 안전망)

## 5. 실시간 분석에서의 대응 관계

실시간 워커(hybrid-engine: `realtime_tracker` + `minsu_detector` + `scoreboard_reader`)도 같은 원칙:

```
스트림 매 프레임 → 탑뷰 검사 ─ 실패 → 스킵 ("지원하지 않는 카메라 화면")
                     ↓ 통과
              공 움직임 추적 ─ 움직이는 중 → 분석 안 함 ("공 정지 대기")
                     ↓ 세 공 정지 ★ 이 순간에만 분석 진입
              배치 확정 → 확률 엔진 호출 → 확률 표시
              (병행: scoreboard_reader 가 수구·득점 판독)
```

- **분석이 실행되는 프레임 = "탑뷰 + 세 공 정지"가 성립한 순간뿐** (3쿠션 확률은 정지 배치에서만 의미 있음)
- 배치 파이프라인과의 차이는 판정 시점뿐: 배치는 다음 샷까지 보고 **사후 확정**, 실시간은 멈춘 순간 **즉시 확률**을 내고 점수판을 흐르는 대로 반영
