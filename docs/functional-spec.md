# CueCast 기능 명세서

## 1. 문서 개요

이 문서는 CueCast의 사용자 기능, 서버 기능, 데이터 수집 기능과 검증 조건을 정의합니다. 구현 기준은 `main` 브랜치의 `YOLO/local_probability_server.py`, `YOLO/cuecast_yolo/`, `src/`, `db/`, `deploy/`입니다.

### 1.1 시스템 목표

CueCast는 다음 세 가지 확률을 하나의 서비스에서 제공합니다.

1. **경기 전 승률:** 선수 누적 기록과 2026 시즌 시작 스냅샷 기반
2. **현재 샷 성공률:** 샷 직전 세 공의 정규화 좌표 기반
3. **실시간 세트 승률:** 경기 전 우위, 현재 점수, 공격 선수와 현재 샷 성공률 기반

### 1.2 주요 사용자

| 사용자 | 목적 |
|---|---|
| 일반 시청자 | 경기 전·경기 중 우위와 샷 난이도 확인 |
| 시연자 | YouTube 영상과 실시간 분석 결과를 함께 발표 |
| 데이터 관리자 | 영상 URL을 큐에 등록하고 샷 데이터를 축적 |
| 개발자 | API, 모델, OCR, DB 적재 상태 점검 |

### 1.3 용어

| 용어 | 정의 |
|---|---|
| 배치 / 포메이션 | 샷 직전 흰 공·노란 공·빨간 공의 위치 |
| 수구 | 현재 공격 선수가 치는 흰 공 또는 노란 공 |
| 정지 배치 | 세 공이 설정 시간 동안 허용 오차 이내에서 움직이지 않은 상태 |
| 턴 | 점수판 이닝 이벤트를 기준으로 확정한 한 번의 공격 구간 |
| 경기 전 승률 | 경기 시작 전 선수 데이터만으로 계산한 승리 확률 |
| 실시간 세트 승률 | 현재 세트의 점수·수구·샷 성공률을 반영한 승리 확률 |
| 성공 라벨 | 점수판 점수 상승으로 확인된 3쿠션 득점 여부 |

## 2. 기능 요구사항

### 2.1 시스템 실행 및 상태

| ID | 기능 | 요구사항 | 완료 조건 | 우선순위 |
|---|---|---|---|---|
| SYS-01 | 로컬 서버 실행 | 사용자는 루트의 `run_cuecast_local.sh`로 UI와 API를 실행할 수 있어야 한다. | `127.0.0.1:8765` 접속 및 `/api/v1/health` 200 응답 | 필수 |
| SYS-02 | 데이터 없는 실행 | RDS 또는 export 연결이 실패해도 가능한 기능은 계속 실행되어야 한다. | 서버가 종료되지 않고 비활성 기능 안내 표시 | 필수 |
| SYS-03 | 상태 확인 | 서버는 샷 데이터 수, 모델 버전과 경기 전 데이터 소스를 반환해야 한다. | health 응답에 `records`, `modelVersion`, `prematchSource` 포함 | 필수 |
| SYS-04 | OCR 실행 프로그램 확인 | 점수판 OCR 전에 운영체제에 설치된 Tesseract 실행 파일을 사용할 수 있어야 한다. | `tesseract --version` 성공 및 실행 경로 인식 | 필수 |
| SYS-05 | 통합 실행 범위 | YouTube 영상 분석과 DB 연동을 포함한 전체 기능은 현재 로컬 서버와 SSH 터널 방식으로 제공해야 한다. | 로컬 실행에서 영상·DB·OCR·확률 화면 동작 | 필수 |

### 2.2 선수 선택 및 경기 전 승률

| ID | 기능 | 요구사항 | 완료 조건 | 우선순위 |
|---|---|---|---|---|
| PRE-01 | 리그 선택 | PBA와 LPBA 선수 목록을 분리해 조회해야 한다. | `league=PBA|LPBA` 필터 적용 | 필수 |
| PRE-02 | 선수 검색 | 선수명과 축약명을 이용해 두 선수를 선택할 수 있어야 한다. | 검색 결과 선택 시 내부적으로 정식 선수명이 저장됨 | 필수 |
| PRE-03 | 동일 선수 방지 | 같은 선수를 A와 B에 동시에 선택할 수 없어야 한다. | 422 또는 사용자 오류 안내 | 필수 |
| PRE-04 | 경기 전 승률 계산 | Elo·통산·시즌·최근·세부 경기력을 결합해야 한다. | A/B 확률 합이 1이고 입력 순서 교환 시 대칭성 유지 | 필수 |
| PRE-05 | 신뢰도 재가중 | 데이터가 부족한 항목은 기본 가중치에 신뢰도를 곱해 재정규화해야 한다. | `componentConfidences`, `finalWeights` 반환 | 필수 |
| PRE-06 | 설명 제공 | 상위 영향 요인과 신뢰도 수준을 제공해야 한다. | `keyFactors`, `confidence`, `formulaNote` 반환 | 선택 |
| PRE-07 | 선수 이미지 | DB에 이미지가 있으면 반환하고, 없으면 UI 대체 표시를 사용해야 한다. | 이미지 200 또는 `image_not_found` 404 | 선택 |
| PRE-08 | 상대 전적 표시 | 상대 전적은 참고 정보로 표시하되 최종 승률에는 반영하지 않는다. | `headToHeadIncludedInProbability=false` | 선택 |

### 2.3 YouTube 영상 연결

| ID | 기능 | 요구사항 | 완료 조건 | 우선순위 |
|---|---|---|---|---|
| VID-01 | URL 분석 | YouTube URL에서 제목, 영상 길이와 소스 종류를 조회해야 한다. | `/api/v1/youtube/info` 정상 응답 | 필수 |
| VID-02 | 시점 분석 | 사용자가 지정한 시점 이전 구간에서 마지막 안정 배치를 찾을 수 있어야 한다. | timestamp와 lookback을 입력해 배치·확률 반환 | 선택 |
| VID-03 | 실시간 시작 | 현재 재생 시점부터 영상 분석 워커를 시작해야 한다. | 상태가 `starting` 또는 `running`으로 변경 | 필수 |
| VID-04 | 정지 | 사용자가 분석 워커를 중지할 수 있어야 한다. | 추가 프레임 분석이 중단되고 상태 반환 | 필수 |
| VID-05 | 시점 동기화 | 브라우저 재생 위치 변화에 맞춰 분석 워커를 이동시킬 수 있어야 한다. | `timestamp_seconds` 반영 | 필수 |
| VID-06 | 수구 수동 지정 | OCR 확정 전 또는 테스트 시 수구를 수동 변경할 수 있어야 한다. | `white` 또는 `yellow`만 허용 | 선택 |
| VID-07 | 접근 거부 처리 | 배포 환경 또는 외부 네트워크에서 YouTube 요청이 거부되면 서버가 원인을 오류 상태로 전달해야 한다. | 워커 종료 대신 `error` 상태와 상세 메시지 반환 | 필수 |
