02도중에 부딪힌 진짜 문제 — 그리고 우회

🚧 캠퍼스망이 AWS로 나가는 5432(DB 포트)를 차단

맥에서 RDS에 직접 붙으려니 계속 타임아웃. 보안그룹을 전체 개방(0.0.0.0/0)해도 안 됐고, 아래 진단으로 원인이 AWS가 아니라 학교 네트워크임이 확정됐습니다.

테스트결과의미
외부 공개 PostgreSQL 5432 (유럽)성공맥의 5432 송신 자체는 됨
AWS 서울 443 (S3)성공AWS엔 443으로는 닿음
AWS 서울 5432 (RDS)타임아웃AWS로 가는 5432만 막힘

해결: 맥은 443으로 S3에만 업로드(통과됨), RDS 적재는 같은 VPC 안의 EC2가 담당 → 막힌 구간을 아예 안 지나감. 공교롭게 앞서 설계한 S3 경유 구조가 그대로 정답이 됐습니다.

05검증된 것

  • ✓영상 추출 — 3개 영상(PBA 팀리그·챔피언십·LPBA)에서 총 24턴 추출, 천 색 3종(파랑·초록·회색) 모두 처리 · 성공/실패 판정이 점수판·턴 교대와 일치
  • ✓맥 → S3 업로드 — 캠퍼스망(443)에서 결과 파일 정상 업로드
  • ✓EC2 → RDS 적재 — S3의 24턴을 PostgreSQL에 upsert 성공 · 재실행해도 중복 없음
  • ✓보안 마무리 — RDS는 VPC 내부(172.31.0.0/16)만 허용, EC2 SSH(22) 완전 차단 · 임시로 열었던 0.0.0.0/0·맥 IP 규칙 모두 제거

06남은 것 — cron 적용 (준비 완료)

딱 한 줄 남았습니다

10분마다 load_to_db.py --all을 돌리는 cron 설정 스크립트를 만들어 S3에 올려뒀습니다. EC2 브라우저 터미널에 아래 한 줄을 붙여넣으면 자동 적재가 시작됩니다 (등록 + 즉시 1회 실행까지).

cd ~/loader && aws s3 cp s3://billiard-turns-mcamp3w/code/setup_cron.sh . --region ap-northeast-2 && bash setup_cron.sh

이후엔 맥에서 추출 → S3 업로드만 하면 10분마다 EC2가 알아서 RDS에 넣습니다. (선택: 맥 쪽 추출도 cron/launchd로 자동화 가능 — 노트북이 켜져 있어야 하는 트레이드오프)
